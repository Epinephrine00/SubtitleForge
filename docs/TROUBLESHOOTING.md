# 끊김 / 소리 안 남 — 원인 정리

## 1. 프레임이 뚝뚝 끊기는 경우 (가능한 원인)

### A. 매 프레임마다 seek
- **현상**: 재생할 때마다 `av_seek_frame(시간)` 호출 → 키프레임 탐색 + 디코더 플러시가 매번 발생.
- **결과**: 한 프레임 디코딩에 50~100ms 이상 걸려, 30fps(33ms/프레임)를 못 따라감 → 끊김.
- **해결**: “현재 위치” 기준으로 **순차 디코딩**(다음 패킷만 읽기). seek는 사용자 시크/위치 크게 바뀔 때만.

### B. 요청 1개만 처리 + 느린 디코딩
- **현상**: 재생 중 16ms마다 `requestFrame(현재프레임)` 호출 → 이전 요청이 덮어씌워짐. 디코더는 “마지막으로 요청된 1프레임”만 처리.
- **결과**: 한 프레임 디코딩이 80ms 걸리면, 그동안 5번 요청이 바뀌어서 항상 “한참 뒤의 프레임”만 디코딩. 화면은 계속 뒤쳐져 보임 → 끊김.
- **해결**: “재생 위치”를 하나 정해 두고, 그 위치부터 **순차로 여러 프레임**을 디코딩해 버퍼에 채움. UI는 버퍼에서 현재 프레임만 꺼내 씀.

### C. 캐시가 재생 중에는 거의 도움 안 함
- **현상**: 재생 시 매 16ms마다 `m_currentFrame`이 바뀌어, 캐시에 해당 프레임이 거의 없음.
- **결과**: 매 틱마다 새 프레임 요청 → 위 A/B와 겹쳐 끊김.
- **해결**: 순차 디코딩 + 앞으로 몇 프레임 미리 채우는 버퍼.

## 2. 소리가 안 나는 경우 (가능한 원인)

### A. 미디어 로드 전에 재생
- **현상**: `loadFile()` 직후 사용자가 재생 버튼을 누르면, `setSource()`만 하고 아직 로드가 끝나지 않았을 수 있음.
- **결과**: `play()`가 무시되거나 재생이 시작되지 않음.
- **해결**: `mediaStatus == LoadedMedia`(또는 BufferedMedia)일 때만 재생 시작하거나, 재생 요청을 “로드 완료 후 재생”으로 큐잉.

### B. Qt 미디어 백엔드 / 코덱
- **현상**: Windows에서 `QT_MEDIA_BACKEND=windows` 사용 시, 특정 mp4/코덱에서 오디오만 안 나오는 경우가 있음. 반대로 FFmpeg 백엔드에서도 Qt 버전에 따라 오디오 버그가 있을 수 있음.
- **결과**: 비디오만 재생되고 오디오 스트림은 무시되거나 디코딩 실패.
- **해결**: 백엔드 전환 시도(환경 변수 제거 또는 `windows`/`ffmpeg` 바꿔 보기), 에러/상태 로그로 원인 확인.

### C. 에러를 처리하지 않음
- **현상**: `QMediaPlayer::error()` 미연결. 로드 실패·코덱 미지원 시 조용히 실패.
- **결과**: 왜 소리가 안 나는지 알 수 없음.
- **해결**: `error()` 연결 후 메시지/상태 출력 또는 사용자에게 안내.

### D. 볼륨/뮤트
- **현상**: 저장된 볼륨이 0이거나, 다른 코드에서 `setMuted(true)` 호출.
- **해결**: 로드/재생 시 `setMuted(false)` 및 볼륨 재적용(이미 시도됨). 설정 저장값 확인.

---

## 3. 이번 수정에서 할 일

1. **프리뷰**: “매 프레임 seek” 제거 → **순차 디코딩** + 필요 시에만 seek. 재생 위치 기준으로 다음 프레임만 읽어 버퍼에 채우고, UI는 버퍼에서 현재 프레임만 사용.
2. **오디오**: `errorOccurred` 연결(상태바 메시지), `LoadedMedia` 이후 재생, 기본 백엔드 사용.

---

## 4. 소리가 안 날 때 시도할 것

- 상태바에 "Audio: ..." 메시지가 뜨면 내용 확인.
- 실행 전 환경 변수: `set QT_MEDIA_BACKEND=windows` 또는 `set QT_MEDIA_BACKEND=ffmpeg` 후 앱 실행.
- 볼륨 슬라이더가 0이 아닌지 확인.
